docker搭建MySQL一组三从服务

前言

MySQL的一组三从配置通常指的是一个主数据库服务器（Master）和三个从数据库服务器（Slaves）。这种配置是MySQL主从复制（Master-Slave Replication）架构的一部分，用于实现数据的高可用性和读写分离。

主服务器（Master）：

  • 主服务器是数据写入的地方，所有的数据变更（如插入、更新、删除）都发生在主服务器上。
  • 主服务器负责将数据变更记录到二进制日志（Binary Log）中。
  • 主服务器可以处理读写操作，但在高并发环境下，为了提高性能，通常只处理写操作。

从服务器（Slaves）：

  • 从服务器用于读取操作，它们从主服务器复制数据，以减轻主服务器的负载。
  • 从服务器通过复制主服务器的二进制日志来同步数据。
  • 从服务器可以配置为只读，这样它们可以处理大量的读请求，而不会干扰到主服务器上的写操作。
  • 在主服务器发生故障时，一个从服务器可以被提升为新的主服务器，以保证服务的连续性。

环境准备：

  • liunx服务器一台
  • docker已安装
  • docker-compose已安装

准备配置文件

docker-compose-mysql.yml 文件

version: '3.0'
services:
  mysql_master:
    image: mysql:latest
    container_name: mysql_master
    restart: always
    network_mode: bridge
    environment:
     - MYSQL_DATABASE=root
     - MYSQL_ROOT_PASSWORD=123456
    ports:
      - 3307:3306
    volumes:
     - /root/mysql/data/master:/var/lib/mysql
     - /root/mysql/conf/master/my.cnf:/etc/mysql/my.cnf


  mysql_slave01:
    image: mysql:latest
    container_name: mysql_slave01
    restart: always
    network_mode: bridge
    environment:
     - MYSQL_DATABASE=root
     - MYSQL_ROOT_PASSWORD=123456
    ports:
      - 3308:3306
    volumes:
     - /root/mysql/data/slave01:/var/lib/mysql
     - /root/mysql/conf/slave01/my.cnf:/etc/mysql/my.cnf


  mysql_slave02:
    image: mysql:latest
    container_name: mysql_slave02
    restart: always

创建映射卷

mkdir -p /root/mysql/data/{master,slave01,slave02,slave03}
mkdir -p /root/mysql/conf/{master,slave01,slave02,slave03}

master主库配置文件

vim /root/mysql/conf/master/my.cnf


[mysqld]
pid-file        = /var/run/mysqld/mysqld.pid
socket          = /var/run/mysqld/mysqld.sock
datadir         = /var/lib/mysql
secure-file-priv= NULL
server-id=1
# Disabling symbolic-links is recommended to prevent assorted security risks
symbolic-links=0

docker-compose命令

容器创建

docker-compose -f docker-compose-mysql.yml up -d

容器停止

docker-compose -f docker-compose-mysql.yml down

在主库中创建同步用户

进入mysql_master容器

docker exec -it mysql_master bash

连接mysql数据库 （密码123456）

mysql -u root -p

创建允许主从复制的用户 test，密码为 test1234

CREATE USER 'test'@'%' IDENTIFIED WITH mysql_native_password BY 'test1234';
GRANT REPLICATION SLAVE ON *.* TO 'test'@'%';

此时的用户没有创建库或表的权限，以root用户登录主库并执行以下命令赋予test用户权限

GRANT ALL PRIVILEGES ON  *.* TO ‘test’@‘%’ IDENTIFIED BY 'test1234’;

查询主库状态

show master status;

在从库中配置主库信息

切换到slave01数据库，执行一下命令

change master to
master_port=3307,
master_user='test',
master_password='test1234',
master_host='192.168.109.100',
master_log_file='binlog.000003',
master_log_pos=879,
get_master_public_key=1;

然后执行一下命令，开启同步

start slave;

执行以下命令查看从库状态

show slave status\G;

出现两个yes就成功了，然后分别在剩下两台重复执行以上命令

⚠️注意

MySQL数据同步是通过binlog 和 Pos与与主库相同，在手动将主库和从库的同步点对齐之前，或从库数据不一致时候，先停止同步stop slave;从新执行（在从库中配置主库信息）步骤即可
