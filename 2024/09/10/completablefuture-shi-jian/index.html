CompletableFuture实践

引言

在Java传统的Future模式里，咱们都知道，一旦开始了一个异步操作，就只能等它结束，无法知道执行情况，也不能手动完成或者取消。而CompletableFuture呢，就像它的名字一样，是可以”完全控制”的Future。它提供了更多的控制，比如可以手动完成，可以处理异常，还可以把多个Future组合起来，进行更复杂的异步逻辑处理。

对于现代Java程序员来说，掌握CompletableFuture是必不可少的。无论是提高程序的响应性能，还是编写更加清晰、更具可读性的代码，它都能大显身手。

基本概念解读

CompletableFuture到底是什么呢？简单来说，它是一种异步编程工具，可以帮助咱们在未来的某个时刻完成一个计算结果。与Future最大的不同是，它可以被显式地完成，意味着咱们可以在任何时候设置它的值。

简单场景描述

假设你正在开发一个在线购物网站，用户在结账时需要输入他们的信用卡信息。为了确保用户输入的信息是安全的，你需要在后台进行一些验证和加密操作。这些操作可能需要一些时间，比如检查信用卡是否有效、加密信用卡信息等。为了提高用户体验，你希望这些操作能够在后台异步进行，而不影响用户输入和页面加载。

 @Test
    void contextLoads() {
        // 异步执行信用卡验证和加密操作
        CompletableFuture<String> creditCardValidationFuture = CompletableFuture.supplyAsync(() -> creditCardValidation());

        // 同时进行其他操作，比如显示加载动画
        System.out.println(displayLoadingAnimation());

        // 等待信用卡验证和加密操作完成
        String result = creditCardValidationFuture.join();

        // 显示验证结果
        System.out.println(result);
    }

    public String creditCardValidation() {
        try {
            System.out.println("开始信用卡验证------");
            TimeUnit.SECONDS.sleep(5);
            return "信用卡验证成功！";
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        return "信用卡验证失败！";
    }

    public String displayLoadingAnimation() {
        try {
            System.out.println("显示加载动画------");
            TimeUnit.SECONDS.sleep(3);
            return "加载动画显示完成！";
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        return "加载动画显示失败！";
    }

显示加载动画------
开始信用卡验证------
加载动画显示完成！
信用卡验证成功！

场景步骤

  1. 异步执行信用卡验证：使用CompletableFuture.supplyAsync方法异步执行creditCardValidation方法，该方法模拟信用卡验证操作，需要5秒钟。
  2. 显示加载动画：在主线程中调用displayLoadingAnimation方法，模拟显示加载动画，需要3秒钟。
  3. 等待验证完成：使用CompletableFuture.join方法等待信用卡验证操作完成，并获取其结果。
  4. 显示验证结果：根据信用卡验证结果，显示相应的信息。

通过这种方式，用户在等待信用卡验证的过程中，可以看到加载动画，从而提高用户体验。同时，信用卡验证操作不会阻塞主线程，提高了系统的响应速度和效率。

常用方法

1.runAsync 和 supplyAsync方法

说明

runAsync 和 supplyAsync 是 Java 中 CompletableFuture 类的静态方法，用于创建并执行异步任务。它们的主要区别在于任务的返回值类型和方法签名的不同。

public static CompletableFuture<Void> runAsync(Runnable runnable)
public static CompletableFuture<Void> runAsync(Runnable runnable, Executor executor)
public static <U> CompletableFuture<U> supplyAsync(Supplier<U> supplier)
public static <U> CompletableFuture<U> supplyAsync(Supplier<U> supplier, Executor executor)

runAsync

  • 用途: runAsync 方法用于执行一个没有返回值的异步任务，它接收一个 Runnable 参数，并返回一个 CompletableFuture<Void>。

  • 执行方式: 异步任务在默认的 ForkJoinPool.commonPool() 中执行，即使用默认的线程池执行。

  • 语法:

    CompletableFuture<Void> future = CompletableFuture.runAsync(() -> {
    // 异步执行的任务
    System.out.println("Async task running");
});

  • 特点:

    • 无返回值: 异步任务没有返回值，因此 CompletableFuture 的泛型类型为 Void。
    • 默认线程池: 异步任务使用默认的 ForkJoinPool.commonPool() 执行。

  • 示例:

    CompletableFuture<Void> future = CompletableFuture.runAsync(() -> {
    try {
        Thread.sleep(2000); // 模拟耗时操作
        System.out.println("Async task completed");
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
});

future.thenRun(() -> System.out.println("Continue with another task")); // 继续执行下一个任务

supplyAsync

  • 用途: supplyAsync 方法用于执行一个有返回值的异步任务，它接收一个 Supplier<U> 参数（其中 U 是返回值的类型），并返回一个 CompletableFuture<U>。

  • 执行方式: 异步任务在默认的 ForkJoinPool.commonPool() 中执行，即使用默认的线程池执行。

  • 语法:

    CompletableFuture<U> future = CompletableFuture.supplyAsync(() -> {
    // 返回一个结果的异步任务
    return someOperation();
});

  • 特点:

    • 有返回值: 异步任务返回一个结果，因此 CompletableFuture 的泛型类型为 U。
    • 默认线程池: 异步任务使用默认的 ForkJoinPool.commonPool() 执行。

  • 示例:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> {
    try {
        Thread.sleep(2000); // 模拟耗时操作
        return 10;
    } catch (InterruptedException e) {
        e.printStackTrace();
        return -1;
    }
});

future.thenAccept(result -> System.out.println("Result: " + result)); // 输出: Result: 10

总结

特性 runAsync supplyAsync
任务类型 Runnable Supplier<U>
返回值 CompletableFuture<Void> CompletableFuture<U>
执行方式 异步任务没有返回值，使用默认线程池执行 异步任务有返回值，使用默认线程池执行
适用场景 当你需要执行一个没有返回值的异步任务时 当你需要执行一个有返回值的异步任务时

选择建议

  • 使用 **runAsync**：当你需要执行一个没有返回值的异步任务，并且只关注任务的执行而不需要处理返回值时。
  • 使用 **supplyAsync**：当你需要执行一个有返回值的异步任务，并且希望获取并处理任务的返回结果时。

2.whenComplete 和 whenCompleteAsync方法

说明

whenComplete 和 whenCompleteAsync 是 Java 中 CompletableFuture 类的方法，用于在异步任务完成后执行一个消费操作，不论任务是否正常完成或者发生了异常。它们的主要区别在于消费操作的执行方式。

public CompletableFuture<T> whenComplete(BiConsumer<? super T,? super Throwable> action)
public CompletableFuture<T> whenCompleteAsync(BiConsumer<? super T,? super Throwable> action)
public CompletableFuture<T> whenCompleteAsync(BiConsumer<? super T,? super Throwable> action, Executor executor)
public CompletableFuture<T> exceptionally(Function<Throwable,? extends T> fn)

whenComplete

  • 用途: whenComplete 方法用于在 CompletableFuture 完成后，执行一个消费操作（BiConsumer）。这个消费操作接收任务的结果或异常作为输入，并且不返回任何结果。

  • 执行方式: 消费操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。

  • 语法:

    CompletableFuture<T> future = ...;

future.whenComplete((result, throwable) -> {
    if (throwable != null) {
        // 处理异常情况
        handleException(throwable);
    } else {
        // 处理正常结果
        handleResult(result);
    }
});

  • 特点:

    • 同步执行: 消费操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。
    • 无返回值: 方法本身不返回任何结果，因为消费操作没有返回值。

  • 示例:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10);

future.whenComplete((result, throwable) -> {
    if (throwable != null) {
        System.out.println("Exception occurred: " + throwable.getMessage());
    } else {
        System.out.println("Result: " + result);
    }
});

whenCompleteAsync

  • 用途: whenCompleteAsync 方法与 whenComplete 类似，也是在 CompletableFuture 完成后执行一个消费操作。不同的是，它允许指定一个 Executor 或使用默认的 ForkJoinPool 来异步执行消费操作。

  • 执行方式: 消费操作异步执行，可以通过指定 Executor 来控制执行的线程池。

  • 语法:

    CompletableFuture<T> future = ...;
Executor executor = ...; // 可选的自定义 Executor

future.whenCompleteAsync((result, throwable) -> {
    if (throwable != null) {
        // 异步处理异常情况
        handleException(throwable);
    } else {
        // 异步处理正常结果
        handleResult(result);
    }
}, executor);

  • 特点:

    • 异步执行: 消费操作异步执行，可以指定 Executor 控制执行线程。
    • 无返回值: 方法本身不返回任何结果，因为消费操作没有返回值。

  • 示例:

    ExecutorService executor = Executors.newFixedThreadPool(2);
CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10);

future.whenCompleteAsync((result, throwable) -> {
    if (throwable != null) {
        System.out.println("Exception occurred: " + throwable.getMessage());
    } else {
        System.out.println("Result: " + result);
    }
}, executor);

总结

特性 whenComplete whenCompleteAsync
执行方式 在完成的 CompletableFuture 的线程中执行消费操作 异步执行消费操作，可以指定 Executor 控制执行线程
消费函数类型 BiConsumer<T, Throwable> BiConsumer<T, Throwable>
返回值 方法本身不返回任何结果 方法本身不返回任何结果
应用场景 当你需要无论任务正常完成还是发生异常都执行消费操作时 当你需要异步执行消费操作，或者控制消费操作的执行线程时

选择建议

  • 使用 **whenComplete**：当你只需要在任务完成后执行一个消费操作，并且不需要异步执行消费操作时。
  • 使用 **whenCompleteAsync**：当你需要异步执行消费操作，或者希望通过指定 Executor 来控制消费操作的执行线程时。

3.thenApply 和 thenApplyAsync方法

说明

thenApply 和 thenApplyAsync 是 Java 中 CompletableFuture 类的两种方法，用于在异步操作完成后，将任务的结果转换为另一个结果。两者的区别主要在于转换操作执行的线程环境。下面是对这两个方法的详细介绍：

public <U> CompletableFuture<U> thenApply(Function<? super T,? extends U> fn)
public <U> CompletableFuture<U> thenApplyAsync(Function<? super T,? extends U> fn)
public <U> CompletableFuture<U> thenApplyAsync(Function<? super T,? extends U> fn, Executor executor)

thenApply 和 thenApplyAsync 是 Java 中 CompletableFuture 类的方法，用于在异步任务完成后对其结果进行转换操作。它们的主要区别在于转换操作的执行方式。

thenApply

  • 用途: thenApply 方法用于在 CompletableFuture 完成后，对任务的结果进行转换操作。它接收一个 Function 函数作为参数，这个函数将任务的结果作为输入，并返回一个新的结果。

  • 执行方式: 转换操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。

  • 语法:

    CompletableFuture<T> future = ...;

CompletableFuture<U> resultFuture = future.thenApply(result -> {
    // 对任务的结果进行转换操作
    return transform(result);
});

  • 特点:

    • 同步执行: 转换操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。
    • 返回值: 返回一个新的 CompletableFuture<U>，其结果是转换操作后的结果。

  • 示例:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10);

CompletableFuture<String> resultFuture = future.thenApply(result -> "Result: " + result);

resultFuture.thenAccept(result -> System.out.println(result)); // 输出: Result: 10

thenApplyAsync

  • 用途: thenApplyAsync 方法与 thenApply 类似，也是在 CompletableFuture 完成后对其结果进行转换操作。不同的是，它允许指定一个 Executor 或使用默认的 ForkJoinPool 来异步执行转换操作。

  • 执行方式: 转换操作异步执行，可以通过指定 Executor 来控制执行的线程池。

  • 语法:

    CompletableFuture<T> future = ...;
Executor executor = ...; // 可选的自定义 Executor

CompletableFuture<U> resultFuture = future.thenApplyAsync(result -> {
    // 异步执行转换操作
    return transform(result);
}, executor);

  • 特点:

    • 异步执行: 转换操作异步执行，可以指定 Executor 控制执行线程。
    • 返回值: 返回一个新的 CompletableFuture<U>，其结果是异步转换操作后的结果。

  • 示例:

    ExecutorService executor = Executors.newFixedThreadPool(2);
CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10);

CompletableFuture<String> resultFuture = future.thenApplyAsync(result -> "Result: " + result, executor);

resultFuture.thenAccept(result -> System.out.println(result)); // 异步输出: Result: 10

总结

特性 thenApply thenApplyAsync
执行方式 在完成的 CompletableFuture 的线程中执行转换操作 异步执行转换操作，可以指定 Executor 控制执行线程
函数类型 Function<T, U> Function<T, U>
返回值 返回一个新的 CompletableFuture<U>，其结果是转换操作后的结果 返回一个新的 CompletableFuture<U>，其结果是异步转换操作后的结果
应用场景 当你需要在任务完成后对其结果进行转换操作时 当你需要异步执行转换操作，或者控制转换操作的执行线程时

选择建议

  • 使用 **thenApply**：当你只需要在任务完成后对其结果进行同步转换操作，并且不需要异步执行转换操作时。
  • 使用 **thenApplyAsync**：当你需要异步执行转换操作，或者希望通过指定 Executor 来控制转换操作的执行线程时。

4.handle 和 handleAsync 方法

说明

handle 和 handleAsync 方法是 Java 中 CompletableFuture 类的方法，用于处理异步任务的结果或异常。它们在处理方式和执行方式上有所不同。

public <U> CompletionStage<U> handle(BiFunction<? super T, Throwable, ? extends U> fn);
public <U> CompletionStage<U> handleAsync(BiFunction<? super T, Throwable, ? extends U> fn);
public <U> CompletionStage<U> handleAsync(BiFunction<? super T, Throwable, ? extends U> fn,Executor executor);

handle

  • 用途: handle 方法用于处理 CompletableFuture 的结果或异常。它接收一个 BiFunction 函数作为参数，这个函数处理任务的结果或异常，无论任务是否正常完成，都会执行该处理函数。

  • 执行方式: 处理操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。

  • 语法:

    CompletableFuture<T> future = ...;

CompletableFuture<U> resultFuture = future.handle((result, throwable) -> {
    if (throwable != null) {
        // 处理异常情况
        return handleException(throwable);
    } else {
        // 处理正常结果
        return handleResult(result);
    }
});

  • 特点:

    • 同步执行: 处理操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。
    • 返回值: 返回一个新的 CompletableFuture<U>，其结果是处理操作后的结果或异常情况。

  • 示例:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10)
        .thenApply(result -> {
            if (result == 10) {
                throw new RuntimeException("Invalid result");
            }
            return result;
        })
        .handle((result, throwable) -> {
            if (throwable != null) {
                return "Handled exception: " + throwable.getMessage();
            } else {
                return "Result: " + result;
            }
        });

resultFuture.thenAccept(result -> System.out.println(result)); // 输出: Handled exception: Invalid result

handleAsync

  • 用途: handleAsync 方法与 handle 类似，也是处理 CompletableFuture 的结果或异常。不同的是，它允许指定一个 Executor 或使用默认的 ForkJoinPool 来异步执行处理操作。

  • 执行方式: 处理操作异步执行，可以通过指定 Executor 来控制执行的线程池。

  • 语法:

    CompletableFuture<T> future = ...;
Executor executor = ...; // 可选的自定义 Executor

CompletableFuture<U> resultFuture = future.handleAsync((result, throwable) -> {
    if (throwable != null) {
        // 异步处理异常情况
        return handleException(throwable);
    } else {
        // 异步处理正常结果
        return handleResult(result);
    }
}, executor);

  • 特点:

    • 异步执行: 处理操作异步执行，可以指定 Executor 控制执行线程。
    • 返回值: 返回一个新的 CompletableFuture<U>，其结果是异步处理操作后的结果或异常情况。

  • 示例:

    ExecutorService executor = Executors.newFixedThreadPool(2);
CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10)
        .thenApply(result -> {
            if (result == 10) {
                throw new RuntimeException("Invalid result");
            }
            return result;
        })
        .handleAsync((result, throwable) -> {
            if (throwable != null) {
                return "Handled exception: " + throwable.getMessage();
            } else {
                return "Result: " + result;
            }
        }, executor);

resultFuture.thenAccept(result -> System.out.println(result)); // 异步输出: Handled exception: Invalid result

说明

thenApply 和 handle 是 Java 中 CompletableFuture 类的方法，用于处理异步任务完成后的结果，它们在处理结果和异常时有一些区别。

thenApply

  • 用途: thenApply 方法用于在 CompletableFuture 完成后，对任务的结果进行转换操作。它接收一个 Function 函数作为参数，这个函数将任务的结果作为输入，并返回一个新的结果。

  • 执行方式: 转换操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。

  • 语法:

    CompletableFuture<T> future = ...;

CompletableFuture<U> resultFuture = future.thenApply(result -> {
    // 对任务的结果进行转换操作

});

  • 特点:

    • 同步执行: 转换操作在当前线程中执行，即在完成的 CompletableFuture 的线程中执行。

        Thread.sleep(2000); // 模拟耗时操作
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    return 10;
});

CompletableFuture<Integer> future2 = CompletableFuture.supplyAsync(() -> {
    try {
        Thread.sleep(1000); // 模拟耗时操作
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    return 20;
});

CompletableFuture<Void> resultFuture = future1.runAfterBothAsync(future2, () -> {
    System.out.println("Both completed");
}, executor);

resultFuture.join(); // 输出: Both completed

总结

特性 runAfterBoth runAfterBothAsync
执行方式 在两个 CompletableFuture 都完成后的线程中执行 异步执行操作，可以指定 Executor 控制执行线程
操作类型 Runnable Runnable
返回值 方法本身返回一个 CompletableFuture<Void> 方法本身返回一个 CompletableFuture<Void>
应用场景 当你只需要在两个 CompletableFuture 都完成后执行一个操作，并且不需要返回任何结果时 当你需要异步执行操作，并且希望控制操作的执行线程时

选择建议

  • 使用 **runAfterBoth**：当你只需要在两个 CompletableFuture 都完成后执行一个操作，并且不需要异步执行操作时。
  • 使用 **runAfterBothAsync**：当你需要异步执行操作，或者希望通过指定 Executor 来控制操作的执行线程时。

13.thenCompose 和 thenComposeAsync方法

说明

thenCompose 和 thenComposeAsync 是 Java 中 CompletableFuture 类的方法，用于处理一个 CompletableFuture 的结果，并且这个结果是另一个 CompletableFuture 的方式。它们的主要区别在于处理方式和执行方式。

public <U> CompletableFuture<U> thenCompose(Function<? super T, ? extends CompletionStage<U>> fn);
public <U> CompletableFuture<U> thenComposeAsync(Function<? super T, ? extends CompletionStage<U>> fn) ;
public <U> CompletableFuture<U> thenComposeAsync(Function<? super T, ? extends CompletionStage<U>> fn, Executor executor) ;

thenCompose

  • 用途: thenCompose 方法用于处理一个 CompletableFuture 的结果，并将其转换为另一个 CompletableFuture。它接收一个返回 CompletableFuture<U> 的函数作为参数，这个函数将当前 CompletableFuture 的结果作为输入。

  • 执行方式: 转换操作在当前线程中执行，即在当前 CompletableFuture 的线程中执行。

  • 语法:

    CompletableFuture<T> future = ...;

CompletableFuture<U> resultFuture = future.thenCompose(result -> {
    // 返回一个新的 CompletableFuture<U>
    return someAsyncOperation(result);
});

  • 特点:

    • 同步执行: 转换操作在当前 CompletableFuture 的线程中执行。
    • 返回值: 返回一个新的 CompletableFuture<U>，其结果是转换操作返回的 CompletableFuture 的结果。

  • 示例:

    CompletableFuture<Integer> future1 = CompletableFuture.supplyAsync(() -> 10);

CompletableFuture<String> resultFuture = future1.thenCompose(result -> {
    return CompletableFuture.supplyAsync(() -> "Result: " + result);
});

resultFuture.thenAccept(result -> System.out.println(result)); // 输出: Result: 10

thenComposeAsync

  • 用途: thenComposeAsync 方法与 thenCompose 类似，也是处理一个 CompletableFuture 的结果，并将其转换为另一个 CompletableFuture。不同的是，它允许指定一个 Executor 或使用默认的 ForkJoinPool 来异步执行转换操作。

  • 执行方式: 转换操作异步执行，可以通过指定 Executor 来控制执行的线程池。

  • 语法:

    CompletableFuture<T> future = ...;
Executor executor = ...; // 可选的自定义 Executor

CompletableFuture<U> resultFuture = future.thenComposeAsync(result -> {
    // 异步执行转换操作，返回一个新的 CompletableFuture<U>
    return someAsyncOperation(result);
}, executor);

  • 特点:

    • 异步执行: 转换操作异步执行，可以指定 Executor 控制执行线程。
    • 返回值: 返回一个新的 CompletableFuture<U>，其结果是异步转换操作返回的 CompletableFuture 的结果。

  • 示例:

    ExecutorService executor = Executors.newFixedThreadPool(2);
CompletableFuture<Integer> future1 = CompletableFuture.supplyAsync(() -> 10);

CompletableFuture<String> resultFuture = future1.thenComposeAsync(result -> {
    return CompletableFuture.supplyAsync(() -> "Result: " + result, executor);
});

resultFuture.thenAccept(result -> System.out.println(result)); // 异步输出: Result: 10

总结

特性 thenCompose thenComposeAsync
执行方式 在当前 CompletableFuture 的线程中执行转换操作 异步执行转换操作，可以指定 Executor 控制执行线程
转换函数类型 Function<T, CompletableFuture<U>> Function<T, CompletableFuture<U>>
返回值 返回一个新的 CompletableFuture<U>，其结果是转换操作返回的 CompletableFuture 的结果 返回一个新的 CompletableFuture<U>，其结果是异步转换操作返回的 CompletableFuture 的结果
应用场景 当你需要对当前 CompletableFuture 的结果执行异步转换操作时 当你需要异步执行转换操作，并且希望控制转换操作的执行线程时

选择建议

  • 使用 **thenCompose**：当你只需要在当前 CompletableFuture 的线程中执行转换操作，并且不需要异步执行转换操作时。
  • 使用 **thenComposeAsync**：当你需要异步执行转换操作，或者希望通过指定 Executor 来控制转换操作的执行线程时。
