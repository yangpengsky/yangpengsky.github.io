前言

在Spring中，AOP（面向切面编程）是一种编程范式，用于将横切关注点（Cross-cutting Concerns）与核心业务逻辑（Core Concerns）分离。横切关注点是那些存在于应用程序各处的功能或需求，它们不属于特定的业务逻辑，例如日志记录、性能监控、安全性检查等。

AOP通过将这些横切关注点抽象为切面（Aspect），然后将它们独立于主业务逻辑织入到应用程序中，以实现代码的模块化和重用性。这种分离和组合的方式使得应用程序的核心逻辑更加简洁和清晰。

在Spring中，AOP提供了一种优雅的方式来实现切面编程。Spring AOP基于动态代理技术和反射机制，允许开发人员通过简单的配置来定义切面，然后将其应用到目标类的特定方法上。

Spring AOP的几种类型

Spring AOP支持以下几种类型的通知（Advice）：

  1. 前置通知（Before Advice）：在目标方法执行前执行。
  2. 后置通知（After Advice）：在目标方法执行后（无论是否发生异常）执行。
  3. 环绕通知（Around Advice）：在目标方法执行前后执行，可以完全控制方法的执行流程。
  4. 返回通知（After Returning Advice）：在目标方法成功执行后执行。
  5. 异常通知（After Throwing Advice）：在目标方法抛出异常后执行。

Pointcut用法

在Spring AOP中，@Pointcut注解用于定义切入点表达式，它决定了哪些方法会被织入切面的通知。execution规则是切入点表达式的一种常用规则，它指定了需要匹配的方法执行。

execution规则的基本语法如下：

@Pointcut("execution(modifiers-pattern? return-type-pattern declaring-type-pattern? method-name-pattern(param-pattern) throws-pattern?)")

这里的各个部分含义如下：

  • modifiers-pattern：指定方法的访问修饰符，如public、private、protected等。可以使用通配符*表示任意修饰符，留空表示所有修饰符。
  • return-type-pattern：指定方法的返回类型。使用全限定类名表示，如java.lang.String。可以使用通配符*表示任意返回类型，留空表示任意返回类型。
  • declaring-type-pattern：指定方法所属的类。使用全限定类名表示，如com.example.MyClass。可以使用通配符*表示任意类，留空表示任意类。
  • method-name-pattern：指定方法名，如login。可以使用通配符*表示任意方法名。
  • param-pattern：指定方法的参数列表。使用全限定类名表示，如java.lang.String。可以使用通配符*表示任意参数类型，留空表示无参数。
  • throws-pattern：指定方法抛出的异常。使用全限定类名表示，如java.lang.NullPointerException。可以使用通配符*表示任意异常类型，留空表示无异常。

execution规则中的每个部分都是可选的，你可以根据需要选择性地使用它们来匹配特定的方法。

以下是一些例子来演示不同类型的execution规则：

  • 匹配所有public方法：

    javaCopy code
@Pointcut("execution(public * *(..))")

  • 匹配com.example包下的所有方法：

    javaCopy code
@Pointcut("execution(* com.example..*.*(..))")

  • 匹配MyClass类中的所有方法：

    javaCopy code
@Pointcut("execution(* com.example.MyClass.*(..))")

  • 匹配返回类型为java.lang.String的所有方法：

    javaCopy code
@Pointcut("execution(String *(..))")

  • 匹配参数为int类型的所有方法：

    javaCopy code
@Pointcut("execution(* *(int))")

自定义Aspect类

package com.yangpeng.springboot_jwt.aspect;

import com.yangpeng.springboot_jwt.entity.UserEntity;
import com.yangpeng.springboot_jwt.response.Result;
import lombok.extern.slf4j.Slf4j;
import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.*;
import org.springframework.context.annotation.Configuration;
import org.springframework.core.annotation.Order;


/**
 * @author yangpeng
 * @version 1.0.0
 * @date 2023年07月26日 16:23:28
 * @packageName com.yangpeng.springboot_jwt.aspect
 * @className ControllerAspect
 */
@Aspect
@Configuration
@Order(1) //设置切面的优先级，数字越小优先级越高
@Slf4j
public class UserControllerAspect {
    //@Pointcut("execution(modifiers-pattern? return-type-pattern declaring-type-pattern? method-name-pattern(param-pattern) throws-pattern?)")

    UserControllerAspect() {
        log.info("=============初始化UserControllerAspect===============");
    }

    @Pointcut("execution(* com.yangpeng.springboot_jwt.controller.*.*(..))")
    public void pointCut() {
//      @Pointcut("execution(* com.example.MyClass.*(..))")
    }

    /**
     * 环绕通知
     */
    @Around("pointCut()")
    public Object aroundAdvice(ProceedingJoinPoint pjp) throws Throwable {
        Result result = null;
        // 获取将要执行的方法名称
        String methodName = pjp.getSignature().getName();
        // 获取执行方法的参数
        Object[] argst = pjp.getArgs();
        if (argst.length > 0) {
            // 获取第一个参数的类型
            String firstType = argst[0].getClass().getName();
            log.info("firstType:{}",firstType);
            if (argst[0] instanceof UserEntity) {
                UserEntity userEntity = (UserEntity) argst[0];
                log.info("user原来的username：{}",userEntity.getUsername());
//                userEntity.setUsername("123456");
            }
        }
        // 执行原方法
        result = (Result) pjp.proceed();
        result.setMsg("返回信息被修改了");

        return result;
    }


    /**
     * 声明该方法是一个前置通知，
     * 方法执行之前执行
     */
    @Before("pointCut()")
    public void before(JoinPoint joinPoint) {

        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();
        log.info("========={}前置通执行=============",methodName);
    }

    /**
     * 声明该方法是一个后置通知，目标方法执行后执行，
     * 就算目标方法抛出异常也会执行
     * 但是后置通知不能访问目标方法返回的结果
     */
    @After("pointCut()")
    public void after(JoinPoint joinPoint) {
        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();
        log.info("========={}后置通知执行=============",methodName);
    }

    /**
     * 在目标方法正常执行完了之后执行，
     * 可以访问到方法的返回值:result
     */
    @AfterReturning(value = "pointCut()", returning = "result")
    public void afterReturning(JoinPoint joinPoint, Object result) {
        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();
        log.info("========={}返回通知通执行=============",methodName);
    }

    /**
     * 在目标方法抛出异常执行,也可以指定出现制定异常时执行，
     * Exception改为需要指定的一场类型
     */
    @AfterThrowing(value = "pointCut()",throwing = "e")
    public void afterThorwing(JoinPoint joinPoint, Exception e) {
    }
}

package com.yangpeng.springboot_jwt.aspect;

import com.yangpeng.springboot_jwt.annotation.RunTimeLog;
import lombok.extern.slf4j.Slf4j;
import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.springframework.stereotype.Component;

/**
 * @author yangpeng
 * @version 1.0.0
 * @date 2023年07月26日 15:45:57
 * @packageName com.yangpeng.springboot_jwt.aspect
 * @className RunTimeLogAspect
 */
@Aspect
@Component
@Slf4j
public class RunTimeLogAspect {
    private long startTime;

    @Before("@annotation(runTimeLog)")
    public void beforeMethod(JoinPoint joinPoint, RunTimeLog runTimeLog) {
        startTime = System.currentTimeMillis();
    }

    @After("@annotation(runTimeLog)")
    public void afterMethod(JoinPoint joinPoint, RunTimeLog runTimeLog) {
        long endTime = System.currentTimeMillis();
        long elapsedTime = endTime - startTime;
        log.info("{} 方法，执行时间: {} 毫秒",joinPoint.getSignature().getName(),elapsedTime);
    }
}

package com.yangpeng.springboot_jwt.controller;

import com.baomidou.mybatisplus.core.conditions.query.QueryWrapper;
import com.yangpeng.springboot_jwt.annotation.PassToken;
import com.yangpeng.springboot_jwt.annotation.RunTimeLog;
import com.yangpeng.springboot_jwt.entity.UserEntity;
import com.yangpeng.springboot_jwt.enums.AppExceptionCodeMsg;
import com.yangpeng.springboot_jwt.exception.AppException;
import com.yangpeng.springboot_jwt.mapper.UserMapper;
import com.yangpeng.springboot_jwt.response.Result;
import com.yangpeng.springboot_jwt.utils.JwtUtils;
import com.yangpeng.springboot_jwt.utils.RedisUtil;
import com.yangpeng.springboot_jwt.utils.SaltMD5Util;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.CrossOrigin;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RestController;

/**
 * @author yangpeng
 * @version 1.0.0
 * @date 2023年06月21日 12:37:31
 * @packageName com.yangpeng.springboot_jwt.controller
 * @className UserController
 * @describe TODO
 */
@CrossOrigin
@RestController
public class UserController {

    @Autowired
    private JwtUtils jwtUtils;

    @Autowired
    private RedisUtil redisUtil;

    @Autowired
    private UserMapper userMapper;

    @Autowired
    private SaltMD5Util saltMD5Util;

    @PostMapping("/login")
    @PassToken
    @RunTimeLog()
    public Result<?> login(@RequestBody UserEntity user) {


        QueryWrapper wrapper = new QueryWrapper();
        wrapper.eq("username", user.getUsername());

        UserEntity userEntity = userMapper.selectOne(wrapper);

        try {
            if (saltMD5Util.verifySaltPassword(user.getPassword(), userEntity.getPassword())) {
                // 生成token
                String token = jwtUtils.createToken(userEntity.getId(), userEntity.getUsername());
                redisUtil.set("token", token, 1800);
                return Result.success(token);
            }
        } catch (NullPointerException e) {
            throw new AppException(AppExceptionCodeMsg.PASSWORD_ERRPR);
        }
        throw new AppException(AppExceptionCodeMsg.PASSWORD_ERRPR);
    }

    @RunTimeLog
    public Object someMethod() {
        // 这里是你的方法逻辑
        for (int i = 0; i < 1000000; i++) {
            // 一些耗时操作1
        }
        return null;
    }
}

测试

Spring AOP不仅支持基于XML的配置方式，还支持基于注解的配置方式，使得AOP的使用更加便捷和灵活。

总结而言，Spring中的AOP是一种实现面向切面编程的技术，允许开发人员将横切关注点与核心业务逻辑分离，并通过简单的配置将切面应用到目标方法中。这种方式可以提高代码的可维护性和可扩展性，同时让核心业务逻辑保持简洁和聚焦。
